# Secret Scanner

Проект для автоматического сканирования файлов на наличие секретов (API ключей, паролей, токенов и т.д.). Система состоит из бэкенда на FastAPI и фронтенда на Next.js, развернутых с помощью Docker.

## Архитектура и структура проекта

### 1. Frontend (Next.js)
Расположен в директории `frontend/`. Использует App Router и Tailwind CSS.

*   **`app/`**: Основные страницы приложения.
    *   `page.tsx`: Главная страница со сканером.
    *   `rules/page.tsx`: Управление правилами сканирования.
    *   `report/[id]/page.tsx`: Просмотр детального отчета о сканировании.
*   **`components/`**: UI-компоненты.
    *   `common/`: Сложные компоненты (загрузчик файлов, таблицы результатов, модальные окна).
    *   `ui/`: Базовые UI-элементы (кнопки, инпуты, карточки).
*   **`api/`**: Логика взаимодействия с бэкендом через Axios.
    *   `axios.ts`: Конфигурация клиента с использованием `NEXT_PUBLIC_API_BASE`.
*   **`shared/types/`**: Общие TypeScript интерфейсы для согласованности данных.

### 2. Backend (FastAPI)
Расположен в директории `backend/`.

*   **`api/`**: Эндпоинты API.
    *   `scan_routes.py`: Логика запуска сканирования и получения отчетов.
    *   `rules_routes.py`: CRUD операции для правил.
*   **`core/`**: Ядро сканера.
    *   `scanner_engine.py`: Основной движок, использующий регулярные выражения и расчет энтропии.
    *   `entropy.py`: Алгоритмы расчета энтропии для поиска случайных строк (ключей).
*   **`rules/`**: Хранилище правил.
    *   `default_rules.yaml`: Системные правила (неизменяемые).
    *   `custom_rules.yaml`: Пользовательские правила, добавленные через UI.
*   **`services/`**: Бизнес-логика.
    *   `rules_manager.py`: Управление загрузкой, сохранением и применением правил.
    *   `report_generator.py`: Логика формирования и хранения отчетов.
*   **`models/`**: Pydantic схемы для валидации данных.

## Взаимодействие компонентов

1.  **Frontend → Backend**: Фронтенд отправляет запросы на бэкенд (по умолчанию `http://localhost:8000`). В Docker-окружении адрес настраивается через переменную окружения `NEXT_PUBLIC_API_BASE`.
2.  **Docker Orchestration**: 
    *   `docker-compose.yml` запускает два сервиса: `frontend` и `backend`.
    *   Сервис `frontend` доступен на порту `3000`.
    *   Сервис `backend` доступен на порту `8000`.
3.  **Persistence**: Кастомные правила сохраняются в `backend/rules/custom_rules.yaml`, что позволяет им сохраняться между перезапусками контейнеров.

## Важные особенности

*   **Next.js Standalone**: Docker-образ фронтенда оптимизирован с использованием режима `standalone`, что значительно уменьшает его размер.
*   **Многоэтапная сборка**: Оба Dockerfile используют multi-stage build для разделения зависимостей сборки и выполнения.
*   **Маскирование**: При отображении найденных секретов в отчетах используется маскирование (например, `sk-ant-api...`), чтобы предотвратить утечку реальных данных в интерфейсе.

## Запуск проекта

Для запуска всей системы выполните команду:

```bash
docker-compose up --build
```

После запуска:
*   Интерфейс: `http://localhost:3000`
*   API документация (Swagger): `http://localhost:8000/docs`
