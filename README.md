# 🔍 Secret Scanner | Команда "NullPointers"

Проект для автоматического сканирования файлов и публичных репозиториев на наличие секретов (API ключей, паролей, токенов и т.д.). Система создана по принципу **Shift-Left DevSecOps** и работает строго локально (Zero-Trace), не оставляя следов в открытом интернете.

Система состоит из бэкенда на FastAPI и фронтенда на Next.js, развернутых с помощью Docker.

---

## 🌟 Ключевые возможности

*   🧠 **Энтропия Шеннона**: Сканер математически выявляет неизвестные Base64 и Hex строки по уровню их "случайности", динамически повышая уровень угрозы (Severity).
*   🌐 **Сканирование GitHub по ссылке**: Бэкенд умеет выкачивать чистый ZIP-архив (без `.git` истории) напрямую из публичных репозиториев, минуя необходимость ручной загрузки архивов.
*   🛡️ **Anti-Crash защита (Двойной лимит)**: Сервер защищен от переполнения памяти — отбивает архивы > 100 МБ на уровне API и пропускает "тяжелые" файлы > 10 МБ внутри движка сканера.
*   🤖 **Умный эвристический фильтр**: Игнорирование бинарных файлов (поиск нулевого байта `\x00`) и автоматический подбор кодировок (UTF-8, cp1251) для предотвращения падения сервера на старых проектах.
*   🎯 **Поддержка `.secretignore` (Игнор-листы)**: Сканер автоматически читает файл `.secretignore` в корне проекта, позволяя разработчикам исключать из проверки тестовые директории (например, `tests/mock_data/`), что радикально снижает количество ложных срабатываний (False Positives).
*   ⚡ **Готово к CI/CD**: В папке `integrations/` лежит готовый шаблон `.gitlab-ci.yml` для автоматической блокировки сборок в корпоративных конвейерах.

## 🏗 Архитектура и структура проекта

### 1. Frontend (Next.js)
Расположен в директории `frontend/`. Использует App Router и Tailwind CSS.

*   **`app/`**: Основные страницы приложения.
    *   `page.tsx`: Главная страница со сканером.
    *   `rules/page.tsx`: Управление правилами сканирования.
    *   `report/[id]/page.tsx`: Просмотр детального отчета о сканировании.
*   **`components/`**: UI-компоненты.
    *   `common/`: Сложные компоненты (загрузчик файлов, таблицы результатов, модальные окна).
    *   `ui/`: Базовые UI-элементы (кнопки, инпуты, карточки).
*   **`api/`**: Логика взаимодействия с бэкендом через Axios.
    *   `axios.ts`: Конфигурация клиента с использованием `NEXT_PUBLIC_API_BASE`.

### 2. Backend (FastAPI)
Расположен в директории `backend/`.

*   **`api/`**: Эндпоинты API.
    *   `scan_routes.py`: Логика запуска сканирования (ZIP и GitHub) и получения отчетов.
    *   `rules_routes.py`: CRUD операции для правил.
*   **`core/`**: Ядро сканера.
    *   `scanner_engine.py`: Основной движок, использующий регулярные выражения и расчет энтропии.
    *   `entropy.py`: Алгоритмы расчета энтропии для поиска случайных строк.
    *   `severity.py`: Логика динамического определения риска и маскировки.
*   **`rules/`**: Хранилище правил.
    *   `default_rules.yaml`: Системные 26+ правил (неизменяемые).
    *   `custom_rules.yaml`: Пользовательские правила, добавленные через UI.
*   **`services/`**: Бизнес-логика.
    *   `rules_manager.py`: Управление загрузкой, сохранением и применением правил.
    *   `file_handler.py`: Распаковка, обход директорий, обработка `.secretignore` и фильтрация бинарников.
    *   `report_generator.py`: Экспорт отчетов (CSV/JSON).

## 🔄 Взаимодействие компонентов

1.  **Frontend → Backend**: Фронтенд отправляет запросы на бэкенд (по умолчанию `http://localhost:8000`). В Docker-окружении адрес настраивается через переменную окружения `NEXT_PUBLIC_API_BASE`.
2.  **Docker Orchestration**: 
    *   `docker-compose.yml` запускает два сервиса: `frontend` и `backend`.
    *   Сервис `frontend` доступен на порту `3000`.
    *   Сервис `backend` доступен на порту `8000`.
3.  **Persistence**: Кастомные правила сохраняются в `backend/rules/custom_rules.yaml`, что позволяет им оставаться в системе даже после перезапуска Docker-контейнеров.

## 🔒 Важные особенности безопасности

*   **Маскирование секретов**: При отображении найденных секретов в отчетах используется жесткое маскирование (например, `AKIA***MPLE`). Это предотвращает утечку реальных ключей через интерфейс оператора или логи системы.
*   **Next.js Standalone**: Docker-образ фронтенда оптимизирован с использованием режима `standalone`, что значительно уменьшает его размер и поверхность атаки.
*   **Многоэтапная сборка (Multi-stage build)**: Оба `Dockerfile` разделяют зависимости сборки и выполнения, оставляя в финальном образе только необходимый минимум.

## 🚀 Запуск проекта

### Вариант 1: Запуск Веб-платформы (Основной)

Вам не нужно устанавливать Python или Node.js. Для запуска всей системы выполните команду в корне проекта:

```bash
docker-compose up --build
```

После запуска:
*   🖥️ **Интерфейс**: [http://localhost:3000](http://localhost:3000)
*   ⚙️ **API документация (Swagger)**: [http://localhost:8000/docs](http://localhost:8000/docs)

### Вариант 2: Использование CLI-утилиты (Для CI/CD)

Система включает в себя полноценный консольный интерфейс для инженеров безопасности. Вы можете запустить CLI-сканер прямо внутри Docker-контейнера, не поднимая веб-сервер.

**Красивый цветной лог в терминале:**
```bash
docker-compose run --rm backend python cli.py ./tests/test_samples
```

**Экспорт отчета в чистый JSON (для автоматизации пайплайнов):**
```bash
docker-compose run --rm backend python cli.py ./tests/test_samples --format json
```
