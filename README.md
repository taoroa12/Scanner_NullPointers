**Secret Scanner** — это комплексный DevSecOps-инструмент для автоматического поиска утечек чувствительных данных (API-ключей, паролей, токенов, приватных ключей) в исходном коде. 

Система спроектирована по принципам **Shift-Left** (предотвращение утечек до попадания в прод) и **Zero-Trace** (строго локальная работа в In-Memory без сохранения данных на диск).

## 🌟 Киллер-фичи (За рамками ТЗ)

Чтобы сделать инструмент применимым в реальном Enterprise-секторе, мы добавили ряд продвинутых функций:

*   🧠 **Энтропия Шеннона:** Сканер математически выявляет неизвестные Base64 и Hex строки по уровню их "случайности", динамически повышая уровень угрозы, даже если ключа нет в словаре регулярных выражений.
*   🌐 **Direct GitHub Scan:** Бэкенд умеет выкачивать чистый срез кода напрямую из публичных репозиториев по ссылке (без тяжелой `.git` истории).
*   🎯 **Поддержка `.secretignore`:** Сканер автоматически читает игнор-листы разработчиков, позволяя исключать тестовые папки (`tests/mock_data/`), что радикально снижает ложные срабатывания (False Positives).
*   🛡️ **Anti-Crash защита:** Сервер защищен от DDoS и переполнения памяти — отбивает архивы > 100 МБ на уровне API, пропускает бинарники (`\x00`) и "тяжелые" файлы > 10 МБ внутри движка сканера.
*   ⚡ **Готово к CI/CD:** В репозитории (папка `integrations/`) лежат готовые шаблоны `.gitlab-ci.yml` и `pre-commit` хуков для автоматической блокировки сборок.

## 🚀 Быстрый старт (Инструкция для проверяющих)

Для запуска вам не нужны установленные Python или Node.js. Вся система запускается в изолированном Docker-окружении.

### 1. Запуск Web-платформы
Склонируйте репозиторий и выполните команду в корневой папке проекта:

```
docker-compose up -d --build
```
После завершения сборки откройте в браузере:

🖥️ Веб-интерфейс (Сюда загружать ZIP): http://localhost:3000
⚙️ API документация (Swagger): http://localhost:8000/docs
(Для тестирования вы можете использовать архивы с уязвимостями из папки backend/tests/test_zip/secret/).

### 2. Использование CLI-утилиты (Для DevOps)
Наш инструмент имеет полноценный консольный интерфейс для встраивания в конвейеры CI/CD. Его можно запустить внутри собранного Docker-контейнера без поднятия веб-сервера.

Для просмотра красивого цветного отчета в терминале:
```
docker-compose run --rm backend python cli.py ./tests/test_samples
```
Для экспорта отчета в чистый JSON (машинное чтение):
```
docker-compose run --rm backend python cli.py ./tests/test_samples --format json
```
### 🏗 Архитектура безопасности
- [x] **Frontend (Next.js):** Оптимизирован с помощью standalone сборки для минимального веса контейнера.
- [x] **Backend (FastAPI):** Асинхронное ядро со строгой валидацией данных через Pydantic V2.
- [x] **Data Persistence:** Базовые правила хранятся в default_rules.yaml. Пользовательские правила пишутся в custom_rules.yaml для сохранения состояния между перезапусками Докера.
- [x] **Data Masking:** Сканер аппаратно маскирует найденные секреты (оставляя только формат AKIA***MPLE). Исходные значения секретов никогда не отдаются на фронтенд и не выводятся в логи, что исключает вторичную утечку данных.
